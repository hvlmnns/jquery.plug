plug.js — an awesome plugin wrapper.

Introduction

Plug.js is a function to create jQuery plugin prototypes on the fly. And the best thing is, once you’ve created a plugin, you'll have global access to it and its methods.

That means you can use the methods of a plugin on completly different objects or plugins, which leads to less code and great readability.

Plugins

Each plugin consists of three elements:

  • name
  • options
  • methods



Note: click execute to see the code in action.

Basic plugin syntax.

execute
$.plug('play',{ // plugin name
  defaults: {
    option: 'defaults',
    methodoption: 'this is a method'
  },
  init: function () { // default function
    this.proto // prototype of your plugin
    this.el,   // javascript object of selected element
    this.$el,  // jQuery object of selected element
    this.opts  // options merged from defaults, passed options and data attributes

    logger('executed "init" of "'+this.proto.name+'"')
    logger(this.opts.option)

    // this.method(); // calling a method
  },
  method: function () { // custom method
    logger(this.opts.methodoption)
    logger('executed "method" of play')
  }
});

logger('created plugin "play"');

Now the plugin is registred.

Calling the plugin

We can call the plugin in different ways:


Calling the plugin.
It will invoke the init function and nothing more.

execute
// call the play plugin

logger('calling "play"');

$('#myElement').play();

Overwriting the default options:

execute
// passing options argument
$('#myElement').play({
  option: 'overwritten defaults'
});

Adding a callback:

execute
// just using a callback
$('#myElement').play(function(){
  logger('callback')
});

Mixing default options overwrite and callbacks:

execute
// first argument options
// second argument callback
$('#myElement').play({
  option: 'mixing stuff'
},function(){
  logger('callback')
});

overwriting default options with html data-attributes:

execute
// The syntax for this is data-pluginname-optionname

// adding data attribute to object
$('#myElement').attr('data-play-option','data attribute overwrite');

Methods

Methods are the children of a plugin. The great thing about them is, that you can call them seperatly on new elements. The way to do this is pretty straigt forward:


calling a method

execute
// calling a method
$('#myElement').play('method')

calling a method with arguments

execute
// calling a method
$('#myElement').play('method',{
  methodoption: 'overwritten defaults'
},function(){
  logger('callback')
})

Modifying

There are four types of modifiers.

  • copy
  • replace
  • append
  • prepend

copying a plugin

execute
// copy play into winning
$.plug.copy('play','winning');
logger('copied "play" into "winning"')

calling the now seperate plugins

execute

logger('calling "play"');
$('#myElement').play();

logger('calling "winning"');
$('#myElement').winning();

replacing a plugin with copy

execute
$.plug('loosing',{
  init: function () {
    // this is a new plugin for sake of demonstration
    // the next plug command will replace it
  }
});

// replacing a plugin with copy
$.plug.copy('play','loosing');

logger('replaced "loosing" with "play"');

// check the console for information
console.log($.pluglib.loosing)


replacing a method

execute
// replace takes three arguments: pluginname, method name, new function 

$.plug.replace('play','init',function () {
  this.method();
});

logger('"init" replaced by "method"');


appending to a method

execute
// append takes three arguments: pluginname, method name, new function 

$.plug.append('play','init',function () {
  logger('appended function');
})


prepending to a method

execute
// prepend takes three arguments: pluginname, method name, new function 

$.plug.prepend('play','init',function () {
  logger('prepended function');
})
