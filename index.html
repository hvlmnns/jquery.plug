Introduction

Plug is a function to create jQuery plugin prototypes on the fly. Once youâ€™ve created a plugin, you have global access to it.

That means you can use Methods of one Plugin in an entirely different one. That leads to less code and better usabilitye of scopes. For having an overview of the used plugins, you can access them in a global library. For any further information please visit: http://hvlmnns.github.io/jquery.plug/

Plugins

Each plugin consists of three elements:

  • name
  • methods
  • options
All those elements can be updated along the process.

Now Lets create a new plugin.

$.plug('newPlugin');

Okay, now we have created a new plugin with the name "newPlugin" in our plugin library. This library can be accessed via: 

$.sockets

The plugin is not doing anything by now. If you execut it on an object like this, it will inform you that the init method is misssing. 

$('#element').newPlugin()

Methods

Since we have a plugin now, we should take a look on how to create methods.

The default method is init. When no method parameter is passed to the call, init gets called.
Since plug.js cant write code for you, you still have to define it yourself. 

$.plug('newPlugin','init',{
    init: function(){

    }
});

Each method will be be a default jquery plugin later. This ensures that we can use the methods on a global scope. 

$.fn.newPlugin.method()

The actual creating syntax is pretty straight forward.
